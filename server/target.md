# 基本框架

你需要使用go的gin框架来实现一个简单的用户管理系统（注意，后续还有其他的服务，这里只需要先实现用户管理，方便我后续进一步开发）。该系统应包含以下功能：

- 用户注册
- 用户登录
- 查看用户信息
- 更新用户信息
- 删除用户

系统中应该有两种用户：

- 普通用户：只能查看和更新自己的信息
- 管理员用户：可以查看、更新和删除所有用户的信息

你需要实现两种用户，并且在后续的api设计中也需要区分这两种用户的权限，有一些api只能管理员用户才能访问。

你需要使用JWT来实现用户的认证和授权。你可以利用一些社区实现的第三方库，请使用最高质量，最广泛使用的库。

# 数据库

你需要使用MySQL数据库来存储用户信息，数据库名为estore。有两个表：

- user
  - id（主键）
  - username
  - is_admin（tinyint unsigned, 0表示普通用户）
- user_auth
  - id（主键，外键）
  - password

数据库在localhost:3306，用户名为root，密码为123。请你使用gin中的config来配置数据库连接。

我需要使用gorm来操作数据库。

# API设计

请你在设计api时，遵循RESTful风格。

# 错误返回处理

请你返回适当的HTTP状态码，并在响应体中包含错误信息。

例如：

```json
{
  "success": false,
  "code": "404",
  "message": "User not found"
}
```
